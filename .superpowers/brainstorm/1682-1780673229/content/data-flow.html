从加载到保存的完整数据流

📥 加载流程

  用户点击行
      │
      ▼
  KnowledgeList.vue
  viewMarkdown(row)
      │
      ├─ dialogVisible = true
      ├─ 发起 GET /api/tools/knowledge-capture/items/:id
      │       │
      │       ▼
      │  返回: { id, title, url, contentMarkdown, ... }
      │       │
      │       ▼
      │  currentMarkdown.value = data.contentMarkdown
      │       │
      │       ▼
      │  ┌─────────────────────┐
      │  │ MarkdownEditor      │
      │  │ v-model="currentMd"  │
      │  │ 左: 编辑区（CM6）     │
      │  │ 右: 预览区（渲染）    │
      │  └─────────────────────┘

📤 保存流程

  Ctrl+S 按下
      │
      ▼
  MarkdownEditor 内 keydown 监听
      │
      ▼
  emit('save')
      │
      ├─ Dialog 中 → KnowledgeList.vue 处理
      │     saveMarkdown() {
      │       PUT /api/tools/knowledge-capture/items/:id
      │       { contentMarkdown: currentMd.value }
      │     }
      │
      └─ 全屏页中 → MarkdownEditPage.vue 处理
            saveMarkdown() {
              PUT /api/tools/knowledge-capture/items/:id
              { contentMarkdown: currentMd.value }
            }

🆕 新增 API

  PUT /api/tools/knowledge-capture/items/:id
  ────────────────────────────────────────
  Content-Type: application/json

  Request Body:
  {
    "contentMarkdown": "# 修改后的 Markdown 内容"
  }

  Response (200):
  {
    "id": 1,
    "title": "深入理解 JavaScript 闭包",
    "contentMarkdown": "# 修改后的 Markdown 内容",
    "updatedAt": "2026-06-05T15:30:00Z"
  }

  Errors:
    404 → { "message": "Knowledge item not found" }
    400 → { "message": "contentMarkdown is required" }

⚡ 技术要点

关注点 方案
XSS 防护 md-editor-v3 内置 DOMPurify 清洗 HTML
Ctrl+S 冲突 preventDefault + emit，不触发浏览器默认
保存反馈 ElMessage.success / error，状态栏显示最后保存时间
弹窗 ↔ 全屏 全屏页返回时 dialog 自动关闭，列表刷新
已修改但未保存 关闭弹窗/离开页面时提示确认（beforeunload / dialog close hook）
